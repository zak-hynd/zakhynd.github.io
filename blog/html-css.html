Topic chosen: "What are the differences between relative, absolute, and fixed positioning?"

Positioning!

The question asks about the differences between three types of positioning, but, from my quick research, it appears that there are five types: Static, Relative, Absolute, Fixed, Sticky.

I’m going to try to demonstrate each of these positioning types by referencing an element that I’ve used on my home blog page. I wanted to change the basic styles for the links to my blog pages, so I put a • before the link text using this class:

.subpage-links a::before { content: "•"; }

But this is what they looked like:

^^The • is right next to the top left corner of the text. This is the default positioning: static.


If I set the positioning to relative and tell it to move left a bit this moves the element relative to that original static position.


If I instead use absolute positioning, this sets the element’s “origin”, it’s “reference position” relative to its nearest positioned “ancestor”... which, in this case, is ???the position of the div class that contains it, minus the padding:


Fixed positioning is fixed relative to the viewport; so when you scroll the elements can appear like they’re floating over the page. Like eye floaters!

Well this is a bit of a failure; Sticky positioning doesn’t appear to affect my wee elements here in any noticeable way. But from what I understand, it allows the positioning to flip-flop between relative and fixed. A good use case of which is the sorts of headings on pages that stay floating until you scroll past a certain point - or simply a nav bar that always stays visible, stuck at the top of the page (the body element)
