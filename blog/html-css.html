Topic chosen: "What are the differences between relative, absolute, and fixed positioning?"

Positioning shenanigans!

The question asks about the differences between three types of positioning, but, from my quick research, it appears that there are five types: Static, Relative, Absolute, Fixed, Sticky.

I though I would try to demonstrate each of these positioning types by referencing an element that I just finished playing with on this website. I had a bunch of links to my wanted to change the basic styles for the links to my blog pages, so I put a bullet point (•) before the link text using this class:

.subpage-links a::before { content: "•"; }

But this is what they looked like:

position: static

The bullet point is right next to the top left corner of the link's text. This is the default positioning: static

If I set the element's position to relative and tell it to move a bit (left: 1rem; top: 1rem), this moves the element relative to that original static position.

position: relative

position: absolute

Here we have absolute positioning: left 1rem; top -1rem.

In this case, the origin/reference point is the top-left corner of this shaded box thing. This box happens to be the nearest "positioned ancestor" of the element. If I didn't give this shaded box a position, then the bullet point would be up near the top of this page!!

position: fixed

Here we have fixed positioning (left 90%; top 95%). Wait, where is the bullet?! Have a look down the bottom-right corner of the screen. This position is fixed relative to the viewport, the browser window; so when you scroll the elements can appear like they’re floating over the page. Like eye floaters! A more useful example of fixed positioning is with things like nav bars (like at the top of this page).

position: sticky

And finally there is sticky positioning.

Keep scrolling...

I had to add a bunch of top padding to this so that it doesn't get completely hidden by the nav bar.

I also couldn't figure out how to make just the bullet sticky - I swear I managed to do it the other day. So I made the whole link sticky instead.

The sticky thing becomes fixed after its normal position has been scrolled past.

This can be useful for things like table headers.

I personally get really annoyed when looking at tabular when the headers aren't sticky! That's such a common problem in Wikipedia, unfortunately.

the type of positioning one chooses to use for an element entirely depends on the circumstances (reeeeaal helpful advice, I know - I feel like a chatbot writing that).
