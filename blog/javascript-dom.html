Why three languages? Why not one?

In the beginning, in the 1980‚Äôs, there was HTML which was used for all the webpage content, providing elements for various headings, links, paragraphs, etc.
CSS was developed in the mid 90‚Äôs. It was designed to separate the page/document styling from the structure, and allowed pages to be styled much more efficiently.
Roughly a year later JS was created to create interactivity within pages.

As web development evolved, these three languages were adapted, the modularity of distinct functionality was useful, and they became the standard of web development, supported by all web browsers.

If, hypothetically, a new language was introduced to replace these three amigos, backwards compatibility would become an issue.

What is the DOM?

The Document Object Model (DOM) is a programming interface that allows programs to manipulate the document‚Äôs (the page‚Äôs) structure, style, and content. The DOM is an Application Programming Interface (API), which is a set of tools and protocols for building and interacting with software applications.
All the initialisms! DOM, API, HTML, CSS, JS, BS, LOL

So the DOM is not part of the JavaScript language, but it does allow JavaScript to interact with the webpage and do its magic. On that note, the DOM can also be used by other programming languages, such as Python.

The DOM is often referred to as a tree structure, a hierarchical tree of nodes. Each node represents an HTML element that can be called on and manipulated by a program/code, usually JavaScript.

Control Flow

Control flow refers to the order in which individual statements, instructions, or function calls are executed in a program. In everyday life, control flow is like following a set of instructions step by step.
For example, making coffee in the morning:
  1. Start: Lazily decide to make a cup of coffee.
  2. Check that I have coffee beans.
    1. If yes, proceed to step 3.
    2. If no, check if I have instant coffee.
      1. If yes, boil water, pour into a mug, add instant coffee, then move to step 6.
      2. If no: cry.
  3. Grind the beans.
  4. Fill the moka reservoir with water.
  5. Add the ground coffee to the bowl thing, assemble the moka pot, heat it on the stove top, wait for it to finish, pour into a mug.
  6. Check if there‚Äôs milk.
    1. If yes, then add milk to mug.
    2. If no, skip.
  7. Check if there‚Äôs brown sugar.
    1. If yes, add as many as energy levels dictate.
    2. If no, skip.
  8. Drink liquid in mug.
It‚Äôs just following a set of instructions, a bit like a recipe (although most recipes don‚Äôt have so many contingency options!)

Loops

Loops allow you to repeat a set of instructions until a certain condition is met. In everyday life, a loop is like repeating a task until it‚Äôs complete. For example, emptying the dishwasher.

  1. Start: Dishwasher cycle ends. Open dishwasher.
  2. Check if there are items in the dishwasher.
    1. If yes, move on to step 3.
    2. If no, close the dishwasher.
  3. Remove an item from the dishwasher, put the item away in its right place, and return to step 2.

What Are Functions?
Functions are fundamental building blocks in JavaScript and other programming languages. A function is a block of code designed to perform a particular task.
An example of a function, in some kinda pseudocode, might be something like: function checkDishwasher(){
  If (dishwasher is closed) {
    openDishwasher()
  } else {
    removeItem()
  }
}

In this function, I‚Äôve got some instructions, which call on other functions. Once I write a function, I can reuse it wherever I like, I don‚Äôt have to rewrite it, and I don‚Äôt have to see all its complexity.

Why are they useful?

  • Reusability: write code once and reuse it multiple times. This reduces redundancy and makes your code more maintainable.
  • Modularity: Functions help break down complex problems into smaller, manageable pieces. Each function can handle a specific task, making your code easier to understand and debug.
  • Abstraction: Functions allow you to hide the implementation details and expose only the necessary interface. This makes your code cleaner and more readable.
  • Scope & Encapsulation: Functions create their own scope, meaning variables declared inside a function are not accessible outside of it. This helps prevent variable name conflicts and keeps code organised.
