JavaScript Fundamentals

What is JS, and how does it fit with HTML and CSS?

Daniel Thé has a good intro video here. Where she uses several useful analogies. Similarly to a country like Belgium, which has 3 official languages (German, French, and Dutch Flemish), the internet has 3 primary languages: HTML, CSS, and Javascript. She calls them the Tres Amigos - the builder (HTML), the artist (CSS), and the “wizard” (JS).

Now I don’t love this analogy, it’s far from perfect (since when do you need to invoke magic to get productive stuff done between a builder and an artist?? A half-decent Product/Project Manager will do!!!) , but the analogy is good enough for the purpose of introducing them at a high level:

  • Hypertext Markup Language (HTML): Structures the content.
  • Cascading Style Sheets (CSS): Styles the content.
  • JavaScript (JS): Adds interactivity and dynamic behaviour.

Why three languages? Why not one?

In the beginning, in the 1980’s, there was HTML which was used for all the webpage content, providing elements for various headings, links, paragraphs, etc.
CSS was developed in the mid 90’s. It was designed to separate the page/document styling from the structure, and allowed pages to be styled much more efficiently.
Roughly a year later JS was created to create interactivity within pages.
As web development evolved, these three languages were adapted, the modularity of distinct functionality was useful, and they became the standard of web development, supported by all web browsers.
If, hypothetically, a new language was introduced to replace these three amigos, backwards compatibility would become an issue.

Now is a good time to segue to the DOM…

What is the DOM?

The Document Object Model (DOM) is a programming interface that allows programs to manipulate the document’s (the page’s) structure, style, and content. The DOM is an Application Programming Interface (API), which is a set of tools and protocols for building and interacting with software applications.
All the initialisms!
DOM, API, HTML, CSS, JS, BS, LOL

So the DOM is not part of the JavaScript language, but it does allow JavaScript to interact with the webpage and do its magic. On that note, the DOM can also be used by other programming languages, such as Python.

The DOM is often referred to as a tree structure, a hierarchical tree of nodes. Each node represents an HTML element that can be called on and manipulated by a program/code, usually JavaScript.
